09 — Motion & Animation

Motion Principles

Animation serves function, not decoration. Every transition communicates cause-and-effect. Timing is tuned for an operational environment where speed matters.

Fade Up
Page entry, modals appearing, notifications. Subtle upward lift signals arrival.
600ms · ease · delay stagger
Scale In
Tooltips, dropdowns, popovers. Spring-eased for a natural, bouncy pop.
400ms · cubic-bezier(.34,1.56,.64,1)
Slide In
Sidebar panels, drawer overlays, filter panels from edge.
500ms · ease · translateX
Easing & Duration Reference
Micro-interactions
150–200ms
Hover states, button presses, checkbox toggles, focus rings
UI Transitions
250–400ms
Dropdowns, tooltips, accordion expand, tab switches
Page Transitions
500–700ms
Route changes, modal open/close, sidebar expand
10 — Accessibility

Accessibility Standards

WCAG 2.1 Level AA compliance across all components. Every interaction is keyboard-navigable and screen-reader friendly.

Contrast Ratios

18.1:1 ✓ AAA
Teal on White
4.6:1 ✓ AA

16.4:1 ✓ AAA
Keyboard Navigation
Navigate elementsTab
Activate buttonsEnter / Space
Close modalsEsc
Table navigationArrow keys
Focus Ring Style
3px teal ring · 2–3px offset · No border-radius change
Screen Reader Support
All interactive elements have descriptive aria-label attributes. Tables include scope headers. Status changes announced via aria-live regions. Color is never the sole indicator.
